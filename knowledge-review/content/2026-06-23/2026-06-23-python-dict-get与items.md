## 4. 参考答案与易错点解析

<details>
<summary>展开查看参考答案与易错点</summary>

### 第 1 题

- 参考答案：笔记把 Python 字典类比成 C++ 的 `std::unordered_map`。它由键值对 `<k, v>` 组成，核心用途是通过 key 查找 value。
- 常见误区：只记住“字典像列表一样能放东西”，但忘了它的访问是基于 key，不是位置索引。
- 误区产生原因：初学时容易把所有容器都混成“存数据的盒子”。
- 正确判断线索：一看到 `dic['Chinese']` 这样的写法，就要意识到访问入口是键。

### 第 2 题

- 参考答案：根据笔记示例，直接索引一个不存在的键会抛出 `KeyError`；而 `get()` 在键不存在时返回 `None`，不会直接报错。
- 常见误区：把 `get()` 当成“只是写法不同”，忽略它在缺失键时的行为差异。
- 误区产生原因：两者都能在键存在时取到值，所以差异只会在边界场景里暴露。
- 正确判断线索：先问自己“这个键可能不存在吗”；如果可能，优先想到 `get()` 的安全返回行为。

### 第 3 题

- 参考答案：笔记推荐用 `items()`。它返回字典的键值信息，因此可以写成 `for k, v in dic.items(): ...` 分别遍历键和值。
- 常见误区：只遍历字典本身，以为就能同时拿到值；或者忘记 `items()` 返回的是键值对而不是单独的 key 列表。
- 误区产生原因：`list(dic)`、`sorted(dic)` 这些例子也出现了，容易让人混淆“遍历键”和“遍历键值对”。
- 正确判断线索：如果循环体里需要同时使用 key 和 value，就优先联想到 `items()`。
