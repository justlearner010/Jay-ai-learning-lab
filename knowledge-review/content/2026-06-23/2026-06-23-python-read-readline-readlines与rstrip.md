## 4. 参考答案与易错点解析

<details>
<summary>展开查看参考答案与易错点</summary>

### 第 1 题

- 参考答案：
  - `read()`：把内容读出来，偏向整体读取。
  - `readline()`：读取一行。
  - `readlines()`：读取多行，通常得到按行组织的结果。
- 常见误区：把三者都当成“只是名字不同，本质一样”。
- 误区产生原因：它们都属于读取方法，但返回内容的粒度不同。
- 正确判断线索：先问自己“我要整个文件、单行，还是多行列表式结果”。

### 第 2 题

- 参考答案：笔记指出 `rstrip()` 可以去掉字符串末尾的换行符 `\\n`，这样在逐行打印时不会因为原始行尾自带换行而多出空行。
- 常见误区：以为多出的空行是 `print()` 本身出错，或者把 `rstrip()` 理解成“删除所有空格”。
- 误区产生原因：没有区分“行内容里自带的换行符”和打印时额外附带的换行。
- 正确判断线索：只要输入源来自逐行读取，就先检查每一行末尾是否已经带 `\\n`。

### 第 3 题

- 参考答案：笔记里的思路是先 `with open(..., "r") as file` 逐行读取，把每行 `rstrip()` 后放进列表，再用 `sorted(names)` 排序，最后循环输出。
- 常见误区：直接把原始行放进列表排序，导致排序和输出里都夹着换行符；或者把写入模式和读取模式混用。
- 误区产生原因：示例里同时出现了打开文件、清洗字符串、排序、输出，步骤容易混在一起。
- 正确判断线索：按“读取 -> 清洗 -> 收集 -> 排序 -> 输出”这个顺序回忆代码流程。
