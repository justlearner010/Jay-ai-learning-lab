## 4. 参考答案与易错点解析

<details>
<summary>展开查看参考答案与易错点</summary>

### 第 1 题

- 参考答案：笔记把模块理解成“把函数装在里面、可被复用的功能单元”。`import` 的作用就是把模块里的能力加载进当前程序，例如导入 `random` 后再调用其中的函数。
- 常见误区：把模块只看成“一个需要背名字的文件”，忽略它解决的是复用问题。
- 误区产生原因：初学时容易只记语法，不去想为什么程序要拆成模块。
- 正确判断线索：一看到 `import xxx`，先问自己“我是要复用哪类现成能力”。

### 第 2 题

- 参考答案：共同点是二者都通过 `import` 被程序使用；主要差别是标准库通常随 Python 提供，而第三方库要先通过 `pip install ...` 安装后再导入。
- 常见误区：以为第三方库和标准库在使用方式上完全不同，或者把 `pip` 和 `import` 混成同一个动作。
- 误区产生原因：安装阶段和代码使用阶段经常挨着出现，容易被记成一件事。
- 正确判断线索：先分清“安装依赖”发生在命令行，“导入模块”发生在代码里。

### 第 3 题

- 参考答案：因为这讲真正想训练的是“不要重复实现基础工具，而是学会查找、导入、组合现成模块”的工程思维。笔记明确把这点总结为“Python 的强大来自生态系统”和“复用代码”。
- 常见误区：把这讲理解成单纯背 `random`、`cowsay` 或 `statistics` 的零散用法。
- 误区产生原因：示例库很多，容易把注意力放在例子本身，而不是背后的复用原则。
- 正确判断线索：如果一个知识点能迁移到以后用 `requests`、`flask`、AI agent 工具链上，那它更可能是这讲真正重要的内容。
