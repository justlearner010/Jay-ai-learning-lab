## 4. 参考答案与易错点解析

<details>
<summary>展开查看参考答案与易错点</summary>

### 第 1 题

- 参考答案：`.strip()` 负责去掉首尾空白，`.split()` 负责把字符串按分隔规则切开，`.join()` 负责把多个字符串重新拼接成一个字符串，`.replace()` 负责把旧子串替换成新子串。
- 常见误区：把这些方法都记成“字符串处理工具”，但说不清输入输出变化。
- 误区产生原因：只记名字，不跟具体数据形态绑定。
- 正确判断线索：先问自己操作前后数据是“一个字符串”还是“字符串列表”。

### 第 2 题

- 参考答案：`.strip()` 处理的是首尾空白，不会把字符串切成列表；`.split()` 会按空格或指定分隔符切分，返回列表。`.join()` 的方向和 `.split()` 相反，它接收一组字符串并按连接符拼回单个字符串。
- 常见误区：把“清洗字符串”和“切分字符串”混成一步，或者记反 `split` / `join` 的方向。
- 误区产生原因：三者经常连续出现，容易在流程里串台。
- 正确判断线索：看到列表就想到 `join`，看到单个长字符串想拆开才想到 `split`。

### 第 3 题

- 参考答案：可以按 `s.strip()` -> `words = s.split()` -> `' '.join(words)` 这样的顺序组织；如果还要替换其中某个词，再在合适位置插入 `.replace()`。
- 常见误区：先 `join` 再 `split`，或者忘了先清理首尾空白。
- 误区产生原因：没有按数据流顺序思考字符串处理步骤。
- 正确判断线索：按“清洗 -> 切分 -> 处理 -> 拼接”的顺序回忆最稳。
